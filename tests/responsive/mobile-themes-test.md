# Responsive Design Test: Mobile Viewports

## Objective
Verify that all themes (Default, Cartoon, Neo-Brutal) render correctly on mobile screen sizes.

## Pre-requisites
- Portfolio application is running locally or deployed
- Browser with device emulation capabilities (DevTools)

## Test Steps

1. Open browser DevTools and activate device toolbar
2. Select a mobile device preset (e.g., iPhone 12, Pixel 5)
3. Navigate to the portfolio website
4. Verify Default theme renders correctly on mobile
5. Switch to Cartoon theme
6. Verify Cartoon theme renders correctly on mobile
7. Switch to Neo-Brutal theme
8. Verify Neo-Brutal theme renders correctly on mobile
9. Test all interactive elements (buttons, links, forms) in each theme
10. Verify that typography is readable in each theme
11. Check that layout elements don't overlap or break
12. Test theme toggle functionality on mobile view

## Expected Results
- All themes render correctly on mobile screen sizes
- Layout remains responsive and usable
- Interactive elements remain accessible
- Typography remains readable

## Pass/Fail Criteria
- [ ] Default theme renders correctly on mobile
- [ ] Cartoon theme renders correctly on mobile
- [ ] Neo-Brutal theme renders correctly on mobile
- [ ] All interactive elements are accessible on mobile
- [ ] Typography is readable in all themes on mobile
- [ ] No layout overlaps or breaks on mobile
- [ ] Theme toggle is accessible and functional on mobile

## Notes
- Pay attention to touch targets for interactive elements
- Check that animations perform well on mobile devices
- Verify that background patterns scale appropriately on mobile
